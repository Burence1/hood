# Hoodwatch
#### {An angular consuming neighborhood custom Api to display neighborhoods, posts and businesses}, {07-June-2021}
#### {By Burens Omondi}
## Description
{Hoodwatch frontend app consumes a python-django developed api endpoints to display neighborhoods, businesses and posts}
## Setup/Installation
* Navigate to the project repository via https://github.com/Burence1/hood.git
* Change your working directory to the clone location of your preference.
* Use the git clone command in the terminal to clone the project to the preferred local machine directory.
* To work on the cloned project, ensure to have any text editors of your choice.}
{Additionally, all the required assets are available in the stated remote repository. All you need is a text editor of your choice}

## Behaviour Driven Development
## Features
* a user can view,update amd delete neighborhoods
* a user can view,update amd delete businesses
* a user can view,update amd delete posts
* a user can view,update amd delete profiles

# Hood

This project was generated with [Angular CLI](https://github.com/angular/angular-cli) version 11.2.4.

## Development server

Run `ng serve` for a dev server. Navigate to `http://localhost:4200/`. The app will automatically reload if you change any of the source files.

## Code scaffolding

Run `ng generate component component-name` to generate a new component. You can also use `ng generate directive|pipe|service|class|guard|interface|enum|module`.

## Build

Run `ng build` to build the project. The build artifacts will be stored in the `dist/` directory. Use the `--prod` flag for a production build.

## Running unit tests

Run `ng test` to execute the unit tests via [Karma](https://karma-runner.github.io).

## Running end-to-end tests

Run `ng e2e` to execute the end-to-end tests via [Protractor](http://www.protractortest.org/).
